# مشروع ServisOffice (إطار عمل Django)

هذا المشروع هو إعادة بناء لهيكل موقع مكتب الخدمات العامة "servisoffice.com" باستخدام إطار عمل Django، مع تطبيق هيكل قاعدة بيانات للخدمات وتعبئة البيانات من الموقع الحي.

## الإعداد والتشغيل (مُحدَّث)

### 1. المتطلبات الأساسية

*   Python 3.x
*   Django 5.2.9 (مُثبَّت)

### 2. تثبيت الاعتماديات

انتقل إلى مجلد المشروع وقم بتثبيت Django والاعتماديات الأخرى (تم تنفيذها بالفعل):

```bash
cd servisoffice_project/servisoffice
sudo pip3 install -r requirements.txt
```

### 3. إعداد قاعدة البيانات وتعبئة البيانات

تم تطبيق الترحيلات (Migrations) وإنشاء جدول الخدمات، وتم تعبئة البيانات الأولية من الموقع الحي:

```bash
# تم تنفيذها
python3 manage.py makemigrations home
python3 manage.py migrate
python3 manage.py populate_services
```

### 4. تشغيل الخادم

يمكنك تشغيل خادم التطوير المحلي باستخدام الأمر التالي:

```bash
python3 manage.py runserver
```

سيصبح الموقع متاحاً على العنوان: `http://127.0.0.1:8000/`

### 5. ملاحظات هامة

*   تم تحديث ملف `home/views.py` لاستخدام نموذج `Service` الجديد.
*   تم تحديث قوالب `index.html` و `services.html` لعرض الخدمات الديناميكية.
*   تم إضافة أمر إدارة مخصص (`populate_services`) لتعبئة البيانات.
*   تم تحديث جميع الروابط المتعلقة بـ "طلب خدمة" و "تواصل معنا" إلى رابط واتساب افتراضي (`https://wa.me/966555555555`).
